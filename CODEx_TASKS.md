# Codex working rules

You must build OnThe1 in phases.

## Rules
- Complete only the requested phase
- Do not skip ahead
- Do not implement later features early
- Keep code modular and production-friendly
- Stop after each phase and wait for testing feedback
- Always explain:
  - what was built
  - which files changed
  - how to test
  - what remains next

## Build order
1. App skeleton and navigation
2. Shared UI system and theme
3. Detect screen UI only
4. Practice screen UI only
5. Learn screen UI only
6. Quick Count logic
7. Find the 1 trainer logic with local practice tracks
8. Detect flow with stubbed results
9. Real microphone/audio integration
10. Polish and bug fixes
